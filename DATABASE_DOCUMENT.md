# RigPro JSA — Database Reference Document

**Database engine:** SQLAlchemy 2.0 (ORM)
**Supported databases:** SQLite (local dev) · PostgreSQL 15 (production)
**Model file:** `backend/app/models/db_models.py`
**Schema initialised in:** `backend/app/main.py` → `Base.metadata.create_all(bind=engine)`

---

## Overview — All Tables

| # | Table name | ORM class | Purpose |
|---|---|---|---|
| 1 | `users` | `UserDB` | Application users and roles |
| 2 | `form_templates` | `TemplateDB` | Reusable inspection/JSA form schemas |
| 3 | `jsa_records` | `JsaRecordDB` | Job Safety Assessment records |
| 4 | `documents` | `DocumentDB` | Uploaded files and documents |
| 5 | `teams` | `TeamDB` | Team groups |
| 6 | `team_members` | `TeamMemberDB` | User-to-team membership |
| 7 | `actions` | `ActionDB` | Corrective / preventive actions |
| 8 | `action_comments` | `ActionCommentDB` | Comments on actions |
| 9 | `issues` | `IssueDB` | Reported safety issues |
| 10 | `issue_comments` | `IssueCommentDB` | Comments on issues |
| 11 | `schedules` | `ScheduleDB` | Recurring inspection schedules |
| 12 | `schedule_occurrences` | `ScheduleOccurrenceDB` | Individual schedule due-dates |
| 13 | `inspection_records` | `InspectionRecordDB` | Inspection answers and results |
| 14 | `notifications` | `NotificationDB` | In-app user notifications |
| 15 | `audit_logs` | `AuditLogDB` | Action audit trail |
| 16 | `assets` | `AssetDB` | Equipment and asset inventory |
| 17 | `contractors` | `ContractorDB` | Contractor company records |
| 18 | `investigations` | `InvestigationDB` | Incident investigations |
| 19 | `headsup` | `HeadsUpDB` | Broadcast site announcements |
| 20 | `credentials` | `CredentialDB` | Worker licenses and certifications |

> **Note:** The database uses no explicit foreign key constraints. Related records are linked by string ID fields (e.g. `user_id`, `issue_id`). There are no SQLAlchemy `relationship()` declarations — joins are done in router/service code.

---

## ID Format

All primary keys are generated by `store.new_id(prefix)` in `backend/app/services/store.py`.

Format: `{prefix}-{12 random hex chars}`

Examples:
- User: `u-admin`, `u-tech`, `u-sup` (seeded), or `usr-a1b2c3d4e5f6`
- JSA: `jsa-a1b2c3d4e5f6`
- Template: `tpl-0`, `tpl-1`, `tpl-2`, `tpl-3` (seeded), or `tpl-a1b2c3d4e5f6`

## Useful Queries

**Find all open high-priority issues:**
```sql
SELECT id, title, site, created_at FROM issues
WHERE status = 'open' AND priority = 'high'
ORDER BY created_at DESC;
```

**Find JSAs pending approval:**
```sql
SELECT id, job_number, boat_name, created_by, created_at FROM jsa_records
WHERE status = 'pending_approval'
ORDER BY created_at ASC;
```

**Find credentials expiring within 30 days:**
```sql
SELECT user_name, credential_type, expiry_date FROM credentials
WHERE expiry_date BETWEEN CURRENT_DATE AND CURRENT_DATE + INTERVAL '30 days'
ORDER BY expiry_date ASC;
```

**Count actions by status:**
```sql
SELECT status, COUNT(*) FROM actions GROUP BY status;
```

**Inspection scores by template:**
```sql
SELECT template_name, AVG(score) as avg_score, COUNT(*) as total
FROM inspection_records
WHERE status = 'completed'
GROUP BY template_name
ORDER BY avg_score DESC;
```
